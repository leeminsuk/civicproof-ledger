# CivicProof Ledger — 3분 시연영상 촬영 가이드

> 2026 오픈소스 개발자대회 결과보고서 제출용. 이 문서 그대로 따라 찍으면 3분 안에 프로젝트의
> 문제의식 → 동작 → 방어력 → 무결성까지 빠짐없이 담긴다. 라이브 데모 화면은
> `https://leeminsuk.github.io/civicproof-ledger/` (또는 로컬 `python3 -m http.server 4173 --directory web`).

## 0. 촬영 전 준비 (5분)

| 항목 | 권장 |
|------|------|
| 화면 녹화 | macOS 화면기록(⇧⌘5) 또는 OBS Studio. **1080p·30fps 이상** |
| 해상도 | 브라우저 창을 1280×800 이상으로. 데모는 860px 이하에서 1열로 접히므로 **넓게** |

> 자동 재생·공격 실행 버튼이 있어 **손을 많이 안 대도 화면이 알아서 움직인다.**

## 1. 촬영 콘티 (총 3:00)

### 컷 1 — 문제 제기 & 첫인상 (0:00 ~ 0:30)

- **화면**: 페이지 최상단(히어로). 배경에 개인정보 입자가 해시 방벽을 통과해 널리파이어로 바뀌는 애니메이션이 흐른다. KPI 4칸(수리·중복·사업·**온체인 개인정보 0필드**)이 보인다.
- **행동**: 그대로 3~4초 보여준 뒤, 마우스로 "온체인 개인정보 필드 = 0"을 가리킨다.
- **내레이션(약 22초)**:
  > "공모전·장학금·복지 바우처는 예산이 한정돼 중복 수혜를 걸러내야 합니다. 그런데 지금은 기관끼리
  > 주민번호·계좌 같은 원본 개인정보를 대조해야 하죠. CivicProof Ledger는 원본을 공유하지 않고,
  > 공개 원장에는 프로그램별 널리파이어와 서명 증빙만 남깁니다. 온체인 개인정보는 0필드입니다."

### 컷 2 — 직접 해보는 원장 시뮬레이터 (0:30 ~ 1:20)

- **화면**: '직접 해보기 · 원장 시뮬레이터' 섹션.
- **행동**: **[▶ 시나리오 자동 재생]** 버튼을 누른다. 5단계가 0.7초 간격으로 자동 재생되며 감사 로그에 카드가 쌓인다.
  - 시민 A → 오픈소스 공모전 (수리)
  - 시민 B → 오픈소스 공모전 (수리)
  - 시민 A → 창업 장학금 (수리, **다른 널리파이어**)
  - 시민 A → 오픈소스 공모전 (**중복차단**, 앰버색)
  - 시민 C → 복지 바우처 (수리)
- **행동 추가**: 재생이 끝나면 시민 A의 공모전 카드와 장학금 카드의 널리파이어 앞자리(`0x61e7…` vs `0xcda9…`)가 **서로 다름**을 마우스로 짚는다.
- **내레이션(약 40초)**:
  > "직접 신청을 발생시켜 보겠습니다. 자동 재생을 누르면 시민들이 여러 사업에 신청합니다.
  > 같은 시민 A가 공모전과 장학금에 신청하면 널리파이어가 서로 달라 사업 간 추적이 불가능합니다.
  > 하지만 시민 A가 같은 공모전에 다시 신청하면, 보시는 것처럼 즉시 중복으로 차단됩니다.
  > 이 모든 판정은 서명 검증과 널리파이어 규칙을 브라우저에서 실제로 실행한 결과입니다."

### 컷 3 — 공격 극장: 레드팀 12종 (1:20 ~ 2:10)

- **화면**: '공격 방어 시연 · 레드팀 코퍼스' 섹션.
- **행동**: **[공격 12종 실행]** 버튼을 누른다. ATK-01부터 12까지 카드가 0.18초 간격으로 '차단'으로 채워지고, 상단 카운터가 **12 / 12**로 올라간다.
- **행동 추가**: 카드 몇 개(예: ATK-03 발급자 키 위조, ATK-12 감사 로그 위조)를 마우스로 짚어 방어 설명을 보여준다.
- **내레이션(약 40초)**:
  > "심사위원이 가질 법한 '공격자가 이렇게 하면요?'라는 질문 12가지를 실행 가능한 시나리오로 만들었습니다.
  > 증빙 위변조, 발급자 키 위조, 만료 재사용, 중복 재등록, 교차 추적, 증명 위조, 감사 로그 위조까지—
  > 지금 이 12종은 화면에서 실제로 Ed25519 서명과 증명 검산을 돌린 결과이고, 12개 전부 차단됩니다.
  > 이 코퍼스는 CI에 들어 있어, 방어가 약해지면 빌드 자체가 깨집니다."

### 컷 4 — 무결성 대시보드 & 장부 조작 (2:10 ~ 3:00)

- **화면**: '무결성 대시보드' 섹션. 게이지가 **100 / 100 EXCELLENT**, 지수 구성 40·30·20·10, Replay-Verify **MATCH**.
- **행동 1**: 게이지와 지수 구성 막대를 한 번 보여준다.
- **행동 2 (하이라이트)**: 우측 상단 **[감사 로그 조작 시뮬레이션]** 토글을 켠다. 즉시:
  - 게이지 100 → **60**, 등급 **WATCH**
  - '감사 일치' 40 → **0**
  - Replay-Verify **DIVERGED**, `EVENT_ORDER_VIOLATION` 코드가 빨갛게 표시
- **행동 3**: 토글을 다시 꺼서 100/100 EXCELLENT로 복구되는 것을 보여준다.
- **내레이션(약 45초)**:
  > "마지막으로 무결성 대시보드입니다. 시민 무결성 지수는 감사 일치·중복 차단·증빙 유효·개인정보 최소화를
  > 40, 30, 20, 10으로 합산한 결정적 점수라, 누구나 공개 데이터만으로 같은 값을 재현합니다. 지금은 100점입니다.
  > 그럼 장부를 조작해 보겠습니다. 이 토글을 켜면 가짜 이벤트가 주입되는데—보시는 것처럼 Replay-Verify가
  > 이벤트 로그로 상태를 다시 계산해 즉시 어긋남을 잡아내고, 지수는 60점 WATCH로 떨어집니다.
  > '장부를 믿지 말고 다시 계산하라'—이것이 CivicProof Ledger의 핵심입니다. 감사합니다."

## 2. 자막 전용 버전(내레이션 없이 찍을 때)

각 컷 시작에 3~4초 풀스크린 자막을 넣고 화면 동작만 보여줘도 된다.

1. `개인정보를 숨긴 채 중복수혜를 검증한다 — 온체인 개인정보 0필드`
2. `직접 해보기: 같은 사업 재신청은 차단, 다른 사업은 널리파이어가 달라 추적 불가`
3. `레드팀 공격 12종을 브라우저에서 실제 실행 → 12/12 차단`
4. `장부를 조작하면? Replay-Verify가 즉시 탐지 · 무결성 지수 100→60`
